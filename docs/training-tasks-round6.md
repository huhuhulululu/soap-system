# 四代理训练任务（Round 6）

## 目标

- 将前 5 轮沉淀为可长期运行的 SOP。
- 完成四角色毕业考核，形成“可持续训练与运营”机制。

## 本轮重点机制

1. SOP 定版（流程、模板、门槛一次性固化）。
2. 四角色毕业考核（理论 + 实操 + 压测）。
3. 月度运营节奏（规则更新、质量回顾、风险收敛）。

## 12 个任务（Round 6）

1. 规则掌握者：发布 `R2.0`，整合 R1.x 所有有效变更并清理临时 hotfix。
2. 编写者：完成 1 个“综合改动任务”（涉及 writer+continue 的一致性）。
3. 检查者：按 R2.0 进行全量核验，输出分级问题清单。
4. 审计员：执行发布前总审计，确认证据链闭环。
5. 规则掌握者：提交“规则退役清单”（废弃规则与替代关系）。
6. 编写者：针对最高优先级问题完成最终修复并补测试。
7. 检查者：执行全回归 + bulk 压测，形成质量报告。
8. 审计员：主持一次“故障桌面演练”（仅流程演练，不改代码）。
9. 四角色联合：完成毕业实操题（限时 2 小时，端到端闭环）。
10. 审计员：给出四角色个人评分与通过/不通过。
11. 规则掌握者：制定下月规则治理计划（新增、变更、审计重点）。
12. 审计员：发布 Round 6 总结与长期运营建议。

## 毕业放行条件

- R2.0 规则集完整、无未归档临时规则。
- 综合改动任务通过审计。
- 回归与压测通过：
  - `cd frontend && npm test`
  - `cd frontend && BULK_SEED_COUNT=100 npx vitest run engine-random`
- 四角色评分均 >= 80/100，且审计员错误放行率为 0。
