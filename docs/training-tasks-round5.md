# 四代理训练任务（Round 5）

## 目标

- 通过异常注入和对抗样本，验证系统在恶劣输入下的规则韧性。
- 训练审计员在“高压+不完美证据”场景下的阻断能力。

## 本轮重点机制

1. 对抗样本集（误导输入、边界输入、冲突输入）。
2. 故障注入（可控）+ 证据缺失场景模拟。
3. 审计红队对检查流程进行逆向挑战。

## 12 个任务（Round 5）

1. 规则掌握者：定义 10 条“高风险规则”并指定攻击面。
2. 编写者：针对 3 条规则构造对抗输入样本（不改规则定义）。
3. 检查者：识别对抗样本违规点并输出证据。
4. 审计员：抽查检查结果，标记漏检/误报。
5. 编写者：在测试环境注入 1 个可控故障（例如字段映射偏移）。
6. 检查者：复现故障并量化影响面。
7. 规则掌握者：发布临时约束补丁（R1.3-hotfix）并说明回收条件。
8. 编写者：按 hotfix 修复并最小化影响范围。
9. 检查者：验证 hotfix 是否引入副作用（单调性/SOA/完整性）。
10. 审计员：发起红队挑战（随机抽 5 条结论反证）。
11. 检查者：回应反证并修订结论（若成立）。
12. 审计员：发布“红队审计报告”（放行/驳回 + 后续改进项）。

## 本轮放行条件

- 对抗样本违规识别率 >= 95%。
- 漏检率 <= 3%，误报率 <= 10%。
- hotfix 后回归全绿：
  - `cd frontend && npm test`
  - `cd frontend && BULK_SEED_COUNT=100 npx vitest run engine-random`
- 红队审计报告完整且可追溯。
